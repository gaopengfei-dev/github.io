# Nuxt

# 简介

Nuxt.js 是基于 vue 的通用应用框架，可用来创建服务端渲染 (SSR) 应用,也可充当静态站点引擎生成静态站点应用,具有优雅的代码结构分层和热加载等特性。 我们都知道 vue.js 是 SPA(单页面应用)，不利于 seo，初次加载的时间较长。不适合做官网类型的项目，Nuxt.js 是 SSR 服务端渲染，和 SPA 相比会带来巨大的 SEO 提升，适用于开发官网和推广类型等项目。

# 优势

1、模块化

Nuxt 基于一个强大的模块化架构。你可以从 50 多个模块中进行选择，让你的开发变得更快、更简单。对 PWA 的支持、添加谷歌分析到你的网页或生成网站地图，这些功能都无需重新发明轮子来获得。

2、高性能

Nuxt.js 默认会优化你的应用程序。尽可能地利用 Vue.js 和 Node.js 的最佳实践来构建高性能的应用程序。Nuxt 帮你把所有不需要的比特都从你的应用程序中剔除，并且还包含了一组分析器，以便更好地优化你的应用程序。

3、服务器端渲染

这是 Nuxt 最受欢迎的模式。利用 SSR（也叫做 "universal" or "isomorphic" 模式），Node.js 服务器将基于 Vue 的组件渲染成 HTML 并传输到客户端，而不是纯 javascript。与传统的 Vue SPA 相比，使用 SSR 将带来巨大的 SEO 提升、更好的用户体验和更多的机会。

因为由开发者独立实现 SSR 将会非常繁琐，Nuxt.js 对此提供了开箱即用的全面支持，并帮你规避常见的陷阱。

4、生成静态站点

Nuxt.js 支持基于 Vue 应用程序生成静态站点。这是“两全其美”的，因为你不要服务器，但是仍能获得 SEO 的好处，这是因为 Nuxt 将预先渲染所有页面，并且包括必要的 HTML。此外，你可以轻松地将生成的页面部署到 Netlify 或 GitHub pages 上。

# 使用方法

# 安装

确保安装了 npx（npx 在 npm5.2.0 默认安装了）运行

npx create-nuxt-app <项目名>

或使用

yarn create-nuxt-app <项目名>

接下来会让你进行一些配置

1、项目名的配置 Project name:(...) 括号中为默认项目名

2、选择语言，可选择 javaScript 或 typeScript

3、选择包管理工具 npm 或 yarn 一般选择 npm。

4、选择 UI 框架

5、接下来选择 axios 作为 nuxt 模块，注意需要按下空格才能选中

6、下面选择代码检查工具，同上需要按空格选中

7、选择测试工具

8、选择 ssr 服务端渲染，spa 为客户端渲染

9、选择部署目标

10、选择部署工具

11、选择代码管理工具

安装完成后 cd 到该项目中，执行

npm run dev

# 目录结构

# 主要目录简介

--layouts 目录 为 Nuxt 的布局目录，用于组织应用的布局组件，在视图文件中配置对应的 layouts:"模版名"，选择对应的模版。

layouts:'info' // 默认为default

--pages 目录为页面目录。Nuxt.js 框架读取该目录下所有的 .vue 文件并自动生成对应的路由配置。Nuxt 是没有对应的路由管理文件的，在生成视图文件时就会生成对应的路由。路由的跳转方法与传参方式与 vue 相同。

pages/info/index.vue的路由为 /info
pages/info/detail/index.vue的路由为 /info/detail
pages/info/test/_id.vue 的路由为 /info/test/:id?

嵌套路由

文件结构

pages/
--| users/
-----| \_id.vue
-----| index.vue
--| users.vue

Nuxt 自动生成的配置文件如下

router: {
    routes: [
        {
            path: '/users',
            component: 'pages/users.vue',
            children: [
                {
                    path: '',
                    component: 'pages/users/index.vue',
                    name: 'users'
                },
                {
                    path: ':id',
                    component: 'pages/users/_id.vue',
                    name: 'users-id'
                }
            ]
        }
    ]
}

注

若无额外配置，layouts，pages 目录不能被重新命名

# Nuxt 引入第三方组件的方法

eg:Nuxt 引入 ElementUI

1、下载 ElementUI 执行 npm i element-ui 。

2、在 plugins 文件夹下创建 elementUI.js 文件

3、填写一下代码

import Vue from 'vue';
import Element from 'element-ui';
import locale from 'element-ui/lib/locale/lang/zh-CN';

Vue.use(Element, { locale });

4、在 nuxt.config.js 文件下的 css 代码块中引入 element 的样式

css: [ 'element-ui/lib/theme-chalk/index.css' ];

5、在 nuxt.config.js 文件下的 plugins 代码块中引入 elementUI.js

plugins: [{ src: './plugins/elementUI.js' }];

# Nuxt 配置 SEO 方法

根结点的可以在 nuxt.config.js 中的 head 里进行配置

Nuxt.js 提供 head 方法用于更新当前页面的头部标签和 html 属性。可使用以下代码对当前也面配置 SEO 信息。

// head与生命周期函数平级
head() {
    return {
      title: this.seoInfoList.seoTitle,
      meta: [
        {
          hid: "description",
          name: "description",
          content: '天津市拓甫网络科技开发有限公司'
        },
        {
          name: "keywords",
          content: '拓甫科技'
        }
      ]
    };
  },

# Nuxt 项目的打包和部署方法

Nuxt 打包和 vue 相同，使用 npm run build 即可，打包完成生成.nuxt（run dev 时也会生成）;

# 部署方法

Nuxt 部署时，先由运维人员在服务器中制定好文件夹，前端将.nuxt、static、nuxt.config.js、package.json 四个文件迁移至服务器中，并执行 npm install 安装依赖。完成后执行 npm start 即可完成部署
